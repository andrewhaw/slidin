Welcome to Slidin

This is a brief introduction on how to set up Slidin' on your own server. To install Slidin', you will require:

  • A server with PHP 5 or higher
  • FTP access
  • An understanding of HTML markup
  • Some patience

Download Slidin'

The first step is to download Slidin'. In that folder you will find a series of files you should upload to your server. Once uploaded, visit the URL you have uploaded the files to and see if the basic template still loads.

For example, if you upload Slidin' to the main level of your website (ie. spatula.ca/slidin/), visit that folder to access the main page.

Common Install Problems

Common issues when installing Slidin' may include:

  • File permission errors; files should be set to 644 and folders to 755
  • .htaccess problems; ensure your server allows mod_rewriting
  • Can't access the index; ensure the folder has uploaded and you can run .PHP files

Slidin' Basics

File structure

Slidin automatically maps any .htm files in the 'content' folder to a URL. For example, if you have a file called 'Test-Presentation.htm', the presentation would be accessible through yoururl.ca/slidin/Test-Presentation and would also be listed along with any other .htm files (in the content folder) on the main page at yoururl.ca/slidin/

Slidin' Basics

Setup

Make sure to edit the 'setup.php' file available in the 'inc' folder to set a standard title for all your slide decks. It also allows you to change the file-type extension used for slide decks (from the default .htm), though you should only consider changing it if you are looking to run custom server-side (ie. PHP) scripts in your decks themselves.

Slidin' Basics

Markup

Markup for Slidin' is like coding a webpage. There are two main pieces of code that are absolutely necessary, and any other code can be injected later on:

<article class="slide-deck"> - Every slide deck must be wrapped in this tag for the deck to work.

<section class="slide"> - Every slide must be wrapped in this tag as otherwise the slide transitioning will not work.

The next slide has an example.

Markup Example

<article class="slide-deck">
    <section class="slide">
        <h1>Slide title</h1>
        <p>Some slide content<p>
    </section>
    <section class="slide">
        <h1>Slide number 2 title</h1>
    </section>
</article>

Slidin' Advanced

Markup

To ensure that your slides can be accessed directly after the fact, add a meaningful id to each slide's <section> tag:
<section class="slide" id="slidin-advanced-markup">

Adding an id means that your slides will each be uniquely accessible through the URL for your slides, in addition to #id (ie. yoururl.ca/slidin/A-Presentation#id)

Slidin' Advanced

Styling and Interaction

To apply custom styles or interactions to specific elements of your slide deck, you can build your own CSS or JS files. Slidin' has been set up to look for a folder and CSS file which share the same name of your slide deck's .htm file.

For example, for 'A-Presentation.htm', Slidin' will look for a folder in 'content' that is called 'A-Presentation'. If there is a 'A-Presentation.css' or 'A-Presentation.js' file in there, Slidin' will automatically apply them to the associated slide deck.

Slidin uses the jQuery library to ease building custom interactions.

Slidin' Advanced

Email encryption

To avoid spam, Slidin' includes a method with which to encrypt your email address. Type your email address in the box below to generate the HTML code below:

<a href="#insertslideid" data-e="" data-subject="Insert Email Message Subject" class="e">Your email link</a>
